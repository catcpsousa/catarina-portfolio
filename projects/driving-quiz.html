Projecto • Sobre Rodas

Plataforma front-end para preparação de exame de código

Aplicação local-first em HTML, CSS e JavaScript que junta quizzes, gamificação, perfil, gestão de questões e revisão guiada numa experiência única para estudar e acompanhar progresso.

Visão geral

O Sobre Rodas foi criado como uma ferramenta prática para preparação da teoria de condução em Portugal. O foco esteve em combinar estudo, simulação, revisão e motivação num só espaço, sem dependência de backend para a experiência principal.

Funcionalidades principais

  • Fluxo de quizzes por categoria A, B, C e D com disponibilidade automática.
  • Modos de quiz múltiplos: simulado de exame, simulação real com tempo, modo livre, estudo com feedback imediato, fraquezas críticas, adaptativo, revisão ultra-rápida e favoritos.
  • Navegação manual entre perguntas durante a sessão, com anterior e seguinte.
  • Classificação de tipos de erro: distração, lacuna de conhecimento e confusão de sinais.
  • Página de perfil com identidade editável, foto, cover, progressão de badge e cartões analíticos.
  • Sistema de gamificação com Rodinhas, desbloqueios, níveis e gastos na loja.
  • Página Loja para gastar Rodinhas em boosts e unlocks úteis.
  • Página Revisão para workflows de revisão pós-sessão e caderno de notas.
  • Header partilhado responsivo com navegação mobile/tablet, ações rápidas e atalho de perfil.
  • Página Admin para gestão de questões, temas, explicações, imagens e estado ativo.

Processo de implementação

A app foi desenhada com um modelo local-first, centrado em `localStorage`, para manter dados de utilizador, histórico, estatísticas, favoritos, revisão e gamificação. A separação de responsabilidades por ficheiros tornou o sistema mais fácil de manter e expandir.

Escolha das cores

A identidade visual foi construída para transmitir foco, energia e clareza durante o estudo. A paleta e os contrastes ajudam a distinguir modos, estados, progresso e destaques sem comprometer a legibilidade em sessões longas.

Documentação e estrutura

O README e a arquitetura de pastas mostram claramente onde cada parte do sistema vive: UI, lógica de perfil, loja, revisão, dados e layout partilhado. Isso facilita manutenção, depuração e futuras melhorias sem mexer em toda a base ao mesmo tempo.

Tecnologias usadas

HTML5 CSS3 JavaScript Bootstrap 5.3.3 jsPDF localStorage

Arquitetura do projeto

1. Base de dados local

Normalização de dados e keys como question bank, profile, history, stats, favorites, reminders e gamification.

2. Layout partilhado

Header responsivo com branding consistente, atalhos rápidos e navegação adaptada a mobile, tablet e desktop.

3. Camadas funcionais

Separação por ficheiros: `main.js`, `profile.js`, `admin.js`, `store.js`, `review-notebook.js`, `data-store.js` e `shared-layout.js`.

4. Evolução contínua

Estrutura preparada para novos modos, métricas, revisões, melhorias de UI e expansão de conteúdos sem reescrever a aplicação.

Desafios e soluções

O maior desafio foi lidar com muitas funções ao mesmo tempo, sem perder consistência entre páginas e sem duplicar lógica. A solução passou por criar um sistema de dados estável, componentes partilhados e uma separação clara entre estudo, perfil, revisão e administração.

Aprendizagens e próximos passos

Este projeto reforçou a importância de pensar em experiência, progresso e motivação como parte do produto, não só como extras visuais. Como evolução futura, pode ganhar sincronização externa, mais estatísticas, novos tipos de revisão e integração com backend.
